# 2026-03-09 - Float Comparisons Added To Lesson

## Context

Пользователь попросил расширить текущий `float`-урок сравнениями `float` с числами, строками и другими типами.

## Key Findings

- Текущий урок хорошо покрывал точность и `epsilon`, но недопокрывал второй опасный слой: смешивание `float` с `int`, `string`, `bool` и `null`.
- Для `float` этого нельзя оставлять на потом, потому что в реальном backend-коде дробные значения часто приходят именно как строки, флаги или пустые значения на границах системы.
- В современном PHP полезно показывать не только классическое `0.1 + 0.2`, но и кейсы вроде `1.0 == 1`, `1.5 == '1.5'`, `0.0 == false`, `0.0 == null`, `1.5 == 'foo'`.

## Decisions

- Расширить `FloatExplorer` отдельным блоком `Сравнения float с другими типами`.
- Обновить lesson pack и answer template под 19 наблюдений вместо 10.
- Усилить критерий прохождения и письменные вопросы, чтобы пользователь явно увидел наложение двух рисков: приближения `float` и type juggling.

## What This Changes

- Урок теперь строит более реалистичную модель поведения `float` в прикладном PHP, а не только в изолированной арифметике.
- Ошибки на границах системы становятся видимыми раньше, до перехода к следующим юнитам.
