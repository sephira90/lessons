# Execution

`execution/` хранит хронологический журнал выполненных разборов и принятых решений.

Что сюда писать:

- что было проанализировано;
- на основании каких артефактов;
- к каким выводам это привело;
- какие решения приняты;
- какие следующие действия из этого следуют.

Что сюда не писать:

- сырые ответы на задания;
- общие размышления без решения;
- дублирование roadmap без новых выводов.
