# 2026-03-09 - Learning Conditions For Unit 1.1

## Context

В `Unit 1.1` упражнение было построено вокруг интерактивного CLI-ввода, но в текущей среде запуска PHP-консоль не даёт надёжного интерактивного режима. Пользователь также не имел явного места для записи ответов и отдельной поверхности для добавления собственных кейсов.

## Key Findings

- Учебный шаг зависел от интерфейса, который фактически не работает в текущей среде.
- Урок не отделял исходный учебный материал от ответов пользователя и его учебного кода.
- Вопрос "где это делать?" показывал, что упражнение сформулировано неоперационно: цель есть, но поверхность действия не определена.

## Decisions

- Зафиксировать в `AGENTS.md` правило о создании условий для обучения, а не только о передаче объяснений.
- Перевести `Unit 1.1` с интерактивного сценария на файловой сценарий: ответы писать в `artifacts/answers/phase-1/unit-1-1-types-and-type-system.md`.
- Выделить отдельный файл `src/Phase1/LearnerTypeExplorerCases.php` для пользовательских кейсов.
- Сохранить `type-explorer` как инструмент проверки фактических результатов без требования интерактивного ввода.

## What This Changes

- Урок теперь совместим с реальной средой выполнения проекта.
- Пользовательские ответы, учебный код и исходные материалы разведены по разным местам.
- Следующие lesson packs должны сразу проектироваться с явными поверхностями: ответы, учебный код, проверка, артефакты.

## Next Actions

- Пользователь фиксирует предсказания и blind reconstruction в `artifacts/answers/phase-1/unit-1-1-types-and-type-system.md`.
- Пользователь добавляет четыре собственных кейса в `src/Phase1/LearnerTypeExplorerCases.php`.
- После этого запускается `.\scripts\composer.cmd type-explorer`, и результаты разбираются причинно, а не только по признаку "верно/неверно".
