# 2026-03-09 - Learning Validation System Added

## Context

После критического разбора мета-анализа модели обучения стало ясно, что в проекте уже есть сильная стратегическая рамка, но не хватает операционного слоя: как именно проверять reconstruction (воспроизведение), transfer (перенос), calibration (калибровку) и capitalization (капитализацию роста).

## Key Findings

- Проект хорошо задаёт глубину, но недостаточно жёстко отделяет реальный рост от ощущения роста.
- Основной пробел был не в отсутствии идей, а в отсутствии управляющего контура с метриками и недельным ревью.
- Без отдельного протокола есть риск, что сильная интеллектуальная среда начнёт переоценивать свою рыночную и инженерную силу.

## Decisions

- Добавить в `AGENTS.md` обязательный протокол переноса, калибровки и капитализации.
- Добавить в `ROADMAP.md` Learning Validation Stack, Weekly Review Protocol и Phase-Based Capitalization Ramp.
- Создать отдельную концептуальную заметку [learning-validation-system.md](/c:/OSPanel/home/lessons.ru/lessons/notes/concepts/learning-validation-system.md).
- Создать шаблон еженедельного ревью [weekly-learning-review-template.md](/c:/OSPanel/home/lessons.ru/lessons/artifacts/reviews/weekly-learning-review-template.md).

## What This Changes

- Проект теперь оценивает не только качество объяснений и кода, но и доказанность переноса, калибровки и постепенной рыночной капитализации.
- Мета-анализ перестаёт быть только красивой диагностикой и начинает влиять на реальный процесс обучения.
- Следующие решения по темпу, углублению и смене фокуса можно принимать по более жёстким сигналам.

## Next Actions

- Использовать weekly learning review в конце первой полной недели работы по Phase 1.
- При разборе каждого следующего юнита явно спрашивать: что воспроизвелось, что перенеслось, чем это калибровано, как это переводится в инженерную ценность.
- Не начинать новый крупный мета-цикл без попытки прожить этот протокол на практике.
