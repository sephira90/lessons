# Big-O Diagnostic

## Инструкция

- На всё отвести 30-45 минут.
- Для каждого ответа укажи сложность по времени и, где уместно, по памяти.
- Если сложность зависит от структуры данных, явно это укажи.

## Задачи

1. Оцени сложность поиска маршрута в массиве из `n` маршрутов, если на каждый запрос framework перебирает их по одному сверху вниз.
2. Оцени сложность такого кода и объясни, что здесь делает операцию дорогой:

```php
$queue = [1, 2, 3, 4, 5];
while ($queue !== []) {
    $item = array_shift($queue);
}
```

3. Оцени среднюю и худшую сложность доступа по ключу в hash table (хеш-таблице). Почему нельзя ограничиться только "O(1)"?
4. Есть цикл по всем контроллерам, а внутри цикл по всем их методам. Если контроллеров `n`, а методов в среднем `m`, какова сложность и где это может стать проблемой в framework bootstrap?
5. Что такое амортизированная сложность и зачем она нужна при анализе динамических структур?
6. Чем различаются:
   - алгоритмическая сложность;
   - цикломатическая сложность;
   - когнитивная сложность?
7. Почему низкая цикломатическая сложность сама по себе не доказывает, что код хороший?
