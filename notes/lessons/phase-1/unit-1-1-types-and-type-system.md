# Unit 1.1 - Типы и система типов

## Цель

Построить причинную модель типов PHP, сравнений и объявлений типов так, чтобы поведение кода сначала предсказывалось, а уже потом проверялось запуском.

## Термины

- `scalar type` (скалярный тип): одиночное значение вроде `int`, `float`, `string` или `bool`.
- `type juggling` (автоматическое приведение типов): преобразование значений, которое PHP делает во время операций, чтобы их можно было сравнить или совместить.
- `strict comparison` (строгое сравнение): сравнение через `===`, где должны совпасть и значение, и тип.
- `strict_types` (режим строгой типизации для объявлений): директива уровня файла, влияющая на то, как проверяются scalar type hints (объявления скалярных типов) аргументов и возвращаемых значений.
- `union type` (объединённый тип): объявление, разрешающее один из нескольких типов, например `int|string`.
- `nullable type` (тип, допускающий `null`): тип, который может принимать `null`, например `?string`.

## Модель

1. В PHP есть значения и типы, но многие операторы допускают неявное преобразование значений.
2. `==` отвечает на вопрос: "могут ли эти значения считаться равными после приведения?"
3. `===` отвечает на вопрос: "совпадают ли здесь и значение, и тип?"
4. `strict_types=1` помогает на границах функций, но не меняет поведение каждого оператора внутри языка.
5. Если ты не можешь предсказать результат сравнения, значит модель выполнения у тебя ещё не под контролем.
