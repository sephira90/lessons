# Unit 1.1a - Float и точность

## Цель

Построить рабочую модель `float` (числа с плавающей точкой), в которой ты различаешь:

- математическое число;
- машинное представление;
- допустимую инженерную погрешность.

Если этой модели нет, то сравнения, деньги, циклы и накопительные вычисления будут казаться "странными", хотя проблема не в PHP, а в ложной интуиции.

## Термины

- `floating point` (число с плавающей точкой): способ хранения вещественных чисел через знак, степень и значащую часть, а не через точную десятичную запись.
- `precision` (точность): сколько значащих цифр система реально удерживает без потери.
- `rounding error` (ошибка округления): расхождение между математическим значением и ближайшим представимым машинным значением.
- `epsilon` (эпсилон, очень малый порог допустимой погрешности): число, через которое удобно проверять "достаточно близко", а не "строго равно".

## Модель

1. `float` хранит приближение, а не точное десятичное значение.
2. Десятичные дроби вроде `0.1`, `0.2`, `0.3` обычно не представимы точно в двоичной форме.
3. Поэтому операции с `float` нужно оценивать не только по математике, но и по машинному представлению.
4. Строгое равенство для `float` часто инженерно бесполезно.
5. Для сравнения дробей обычно нужен допуск через `epsilon`.
6. Для денег `float` почти всегда плохой выбор; обычно нужны `int` в минимальных единицах или специальные десятичные инструменты.
7. Когда `float` сравнивается с `int`, `string`, `bool` или `null`, к проблеме приближения добавляется ещё и type juggling (автоматическое приведение типов).
8. Поэтому `float` опасен не только в арифметике, но и на границах системы, где данные приходят как строки, флаги или пустые значения.

## Обязательные упражнения

1. Предскажи результаты всех наблюдений в `float-explorer`.
2. Ответь письменно:
   - почему `(0.1 + 0.2) === 0.3` даёт `false`;
   - почему `==` здесь тоже не спасает;
   - почему сравнение через `epsilon` полезнее строгого сравнения;
   - почему `0.1 * 10 === 1.0` может быть `true`, но это не опровергает проблему `float`;
   - почему `1.0 == 1` даёт `true`, а `1.0 === 1` даёт `false`;
   - почему `0.0 == false` и `0.0 == null` особенно опасны.
3. Напиши свою функцию:

```php
function almostEqual(float $left, float $right, float $epsilon = 1e-12): bool
{
    return abs($left - $right) < $epsilon;
}
```

4. Ответь на инженерный вопрос: почему деньги нельзя бездумно хранить во `float`?
